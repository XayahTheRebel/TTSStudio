# AGENTS.md

## Collaboration Defaults

- On new projects, initialize Git before starting feature work unless the user explicitly says not to.
- After each completed user request that changes the project, create a Git commit with a clear message.
- If the user explicitly says not to commit for a specific change, skip the commit for that change only.

## Commit Style

- Prefer short, descriptive commit messages that summarize the user-facing change.
- Do basic verification before committing when practical.
